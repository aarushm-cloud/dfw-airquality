# DFW Air Quality — Codebase Architecture

A complete file-by-file map of the repo. Intended as a reference document for a
codebase reorganization / refactor pass. **Every file is listed**, including
generated artifacts and gitignored items.

Legend:
- 🚫 = gitignored (file or directory not tracked in git)
- 📁 = directory
- 📄 = source file
- 🗄️ = data / artifact
- 🖼️ = image / binary asset
- 🧹 = empty file (placeholder; 0 lines)

Line counts are from `wc -l` on the latest version of each file (Python + TS/TSX only).


## 13. Notable inconsistencies / refactor candidates

These are flagged for the refactor pass — not action items, just things to consider:

1. **`ml/analysis/output/output/`** — double-nested directory, almost certainly an accident.
2. **`data/.cache/dfw_highways.pkl` is tracked** — sits in a `.cache/` directory but is committed. Either intentional (it's an expensive build) or an oversight.
3. **`ml/data/.checkpoints/*.parquet` are tracked** — 27 sensor checkpoints committed. They're rebuildable from the training run, so they may not need to be in git.
4. **`data/dashboard_snapshots.csv` is tracked** — a runtime accumulator file, modified every collector run. May want to be gitignored.
5. **`ml/predictor.py`** — flagged as dead code in `CLAUDE.md`; not wired up anywhere.
6. **Empty placeholder files** — `engine/router.py`, `data/ingestion/osm.py`, `viz/charts.py`, `utils/cache.py`. Either implement or remove.
7. **`utils/`** — entire directory contains only an empty `cache.py`.
8. **EPA correction is duplicated** in `data/ingestion/purpleair.py` and `ml/training/collect_training_data.py`. CLAUDE.md notes a planned extraction to `data/corrections.py`.
9. **`ml/research/`** — five scripts representing prior failed experiments. Could be promoted to a single `ml/research/README.md` summary + archive, or pruned.
10. **`design/` filenames have spaces and parentheses** — `DFW Air Quality Dashboard.html`, `index (3).html`, `scene (3).js`. Path-fragile.
11. **`api/openapi.snapshot.json`** is committed — useful as a regression check, but worth confirming the snapshot script is part of CI/the dev loop.
12. **`api/`, `engine/`, `data/`, `ml/training/` all import `config.py` at the top level** — refactor will need to preserve that import shape unless `config.py` moves into a package.
13. **Two parallel UIs** during Phase 6: Streamlit (`app.py` + `viz/heatmap.py`) and React (`web/`). CLAUDE.md mandates Streamlit stays untouched until the React app fully replaces it.
